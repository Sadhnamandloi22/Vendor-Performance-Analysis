## 📌 Project Overview

Retail and distribution businesses purchase inventory from dozens (sometimes
hundreds) of vendors — but not every vendor relationship is equally profitable.
This project analyzes a real-world inventory and sales dataset to answer a core
business question:

> **Which vendors and brands are actually driving profit, and where is the
> business losing money, tying up capital, or missing pricing opportunities?**

The project covers the **full analytics pipeline**: raw CSV ingestion into SQL
Server, SQL-based aggregation and feature engineering, exploratory data analysis
and hypothesis testing in Python, and a final interactive Power BI report for
business stakeholders.

---

## ✨ Features

- 🔄 **Automated data ingestion** — Python script that loads all raw CSVs into SQL
  Server tables with logging and execution-time tracking
- 🧮 **SQL-driven vendor summary** — a multi-CTE SQL query joins purchases, sales,
  pricing, and freight data into a single vendor+brand performance table
- 🧹 **Data cleaning & feature engineering** — computes Gross Profit, Profit
  Margin %, Stock Turnover, and Sales-to-Purchase Ratio per vendor/brand
- 📈 **Exploratory Data Analysis** — distribution analysis, outlier detection, and
  a full correlation heatmap across all numeric KPIs
- 🧪 **Statistical hypothesis testing** — two-sample t-test comparing
  high-performing vs. low-performing vendor segments
- 🏆 **Vendor concentration (Pareto) analysis** — identifies what % of total
  purchase spend is concentrated in the top vendors
- 🎯 **Promotional opportunity detection** — flags brands with low sales but high
  profit margin using percentile-based thresholds (candidates for pricing/promo
  action)
- 📊 **Interactive Power BI dashboard** — 5-page report with KPI cards, Pareto
  charts, opportunity scatter plots, profitability correlation analysis, and an
  operations view (freight, payment cycle, unsold inventory)

## 🗂️ Dataset Information

The dataset is a real-world retail inventory and sales dataset (sourced from
Kaggle) representing transactions for an alcohol/beverage distribution business.
It consists of seven related CSV files:

| File | Description | Grain |
|---|---|---|
| `begin_inventory.csv` | Inventory on hand at the start of the period | Store + Brand |
| `end_inventory.csv` | Inventory on hand at the end of the period | Store + Brand |
| `purchases.csv` | Individual purchase order transactions | Transaction line |
| `purchase_prices.csv` | Vendor list price per brand | Brand |
| `vendor_invoice.csv` | Invoice-level freight, quantity, and payment dates | Invoice |
| `sales.csv` | Individual sales transactions | Transaction line |
| `vendor_sales_summary` | **Derived table** — aggregated Vendor + Brand performance summary (purchases, sales, freight, profit metrics) | Vendor + Brand |

`vendor_sales_summary` is not a raw file — it's generated by joining and
aggregating the tables above via SQL (see `get_vendor_summary.py`), and is the
primary table used for analysis and the Power BI report.

## 🔄 Project Workflow

```
 1. Raw CSV Files
        │
        ▼
 2. ingestion.py  →  Loads CSVs into SQL Server (inventory_db) via SQLAlchemy
        │
        ▼
 3. get_vendor_summary.py  →  Runs a multi-CTE SQL query joining purchases,
        │                     purchase_prices, sales, and vendor_invoice;
        │                     computes Gross Profit, Profit Margin %, Stock
        │                     Turnover, and Sales/Purchase Ratio; writes the
        │                     result back to SQL Server as vendor_sales_summary
        ▼
 4. Exploratory_data_Analysis.ipynb  →  Initial data profiling, column checks,
        │                              missing values, datatype validation
        ▼
 5. vendor_performance_analysis.ipynb  →  Outlier analysis, correlation heatmap,
        │                                hypothesis testing (t-test), Pareto
        │                                analysis, promotional opportunity
        │                                detection
        ▼
 6. Power BI Report  →  Live SQL Server connection, star-schema data model,
                         DAX measures, 5-page interactive dashboard
```

**Step-by-step:**

1. **Ingestion** (`ingestion.py`) — reads every CSV from a local data folder and
   loads it into SQL Server as a table, with logging of execution time.
2. **Vendor Summary** (`get_vendor_summary.py`) — runs a SQL query (using CTEs for
   `FreightSummary`, `PurchaseSummary`, and `SalesSummary`) to build a single
   Vendor + Brand performance table, cleans the data (handles nulls, infinities,
   string trimming), and engineers profitability/efficiency metrics.
3. **EDA** (`Exploratory_data_Analysis.ipynb`) — validates table structures, checks
   for inconsistent vendor name formatting, and confirms data types before deeper
   analysis.
4. **Vendor Performance Analysis** (`vendor_performance_analysis.ipynb`) —
   statistical analysis: correlation heatmap across all KPIs, a t-test comparing
   vendor segments, Pareto/concentration analysis of purchase spend, and
   percentile-based flagging of promotional opportunity brands.
5. **Power BI Dashboard** — connects live to SQL Server, builds a star schema
   (`dim_Vendor`, `dim_Brand`, `dim_Date`), and presents the findings across 5
   interactive report pages.

## 📈 Results / Key Insights

- **Vendor concentration**: a small number of top vendors account for a
  disproportionately large share of total purchase spend — a classic Pareto (80/20)
  pattern, visualized with a cumulative contribution chart.
- **Purchase price ≠ revenue driver**: correlation analysis shows purchase price
  has little to no relationship with total sales or profit — pricing alone doesn't
  explain performance differences between brands.
- **Margin compression at higher price points**: brands with higher average sales
  prices tend to show *lower* profit margins, not higher.
- **Efficient inventory turnover overall**: purchase quantity and sales quantity
  move almost in lockstep across the dataset, indicating the business isn't
  systematically over- or under-buying.
- **Stock turnover is a weak profitability signal**: fast-moving inventory doesn't
  reliably correlate with higher margins — turnover and profitability are largely
  independent.
- **Hidden promotional opportunities**: a statistically-flagged subset of brands
  combine *low sales volume* with *high profit margin* — strong candidates for
  promotional pricing or increased marketing investment, since the margin upside is
  already there.
- **Statistically significant performance gap**: a two-sample t-test confirms that
  high-performing and low-performing vendor segments differ significantly in key
  metrics, not just by chance.

## ⚙️ Installation

### Prerequisites
- Python 3.10+
- Microsoft SQL Server (local or remote instance)
- ODBC Driver 17/18 for SQL Server
- Power BI Desktop (for viewing/editing the report)

### Setup

```bash
# 1. Clone the repository
git clone https://github.com/<your-username>/vendor-performance-analysis.git
cd vendor-performance-analysis

# 2. Create a virtual environment
python -m venv venv
source venv/bin/activate        # On Windows: venv\Scripts\activate

# 3. Install dependencies
pip install pandas sqlalchemy pyodbc numpy scipy matplotlib seaborn jupyter
```

### Database Configuration

Update the SQL Server connection string in `ingestion.py` and
`get_vendor_summary.py` to match your environment:

```python
engine = create_engine(
    "mssql+pyodbc://<SERVER_NAME>/<DATABASE_NAME>?"
    "driver=ODBC+Driver+18+for+SQL+Server&trusted_connection=yes&TrustServerCertificate=yes"
)
```

Place the raw CSV files in the folder referenced by `load_raw_data()` in
`ingestion.py`, or update the `folder_path` variable to point to your local data
directory.

## ▶️ Usage

```bash
# Step 1 — Ingest raw CSVs into SQL Server
python ingestion.py

# Step 2 — Build the vendor_sales_summary table
python get_vendor_summary.py

# Step 3 — Run the analysis notebooks
jupyter notebook Exploratory_data_Analysis.ipynb
jupyter notebook vendor_performance_analysis.ipynb
```

### Viewing the Power BI Report

1. Open `Vendor_Performance_Report.pbix` in Power BI Desktop
2. Update the SQL Server connection (Home → Transform Data → Data Source Settings)
   to point to your own `inventory_db` instance
3. Click **Refresh** to pull live data
4. Use the slicers (Vendor, Classification, Date) to explore the report —
   navigate between the 5 pages using the tabs at the bottom
